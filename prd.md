# 阿川 Blog PRD

## 1. 项目背景

阿川 Blog 是阑梦清川未来五年持续建设的个人博客网站，也是个人在互联网上的长期门面。

网站不以求职为唯一目标，也不应该呈现出过重的求职风格。它首先是为作者自己而设计的个人品牌与内容沉淀空间，求职、合作、交流只是它可能带来的结果。

阿川 Blog 希望传递的核心身份是：阑梦清川是一名 AI 爱好者和内容创作者。

访问者看完网站后，应该形成这样的核心评价：这个人是一个很可信的 AI 深度用户、AI Agent 的资深用户，并且拥有很强的持续学习能力。

## 2. 本次目标

本次目标是定义并搭建阿川 Blog 的第一版网站结构，让它能够清晰承载个人介绍、个人背景、成长路径、作品集和内容创作。

第一版网站需要做到：

- 让访问者快速知道“阿川 Blog”是什么。
- 让访问者理解阑梦清川是一名 AI 爱好者和内容创作者。
- 让访问者看到作者的学习经历、成长路径和个人作品。
- 让访问者能判断作者在 AI Agent、AI 实践和持续学习方面的可信度。
- 为未来五年的内容扩展保留清晰结构。

## 3. 用户和关键场景

### 目标用户

主要用户是在互联网上对博主本人、博主所做的事情、AI 实践、作品集或文章感兴趣的人。

这些用户包括：

- 对阑梦清川本人感兴趣的人。
- 对 AI Agent、AI 实践案例、读书笔记感兴趣的读者。
- 想了解作者作品、项目、小工具和文章的人。
- 潜在合作者、朋友、同行、招聘方或其他想进一步了解作者的人。

### 关键场景

1. 了解作者  
   访问者希望快速知道阑梦清川是谁、现在关注什么、为什么值得继续了解。

2. 阅读内容  
   访问者希望阅读 AI Agent、读书笔记、AI 实践案例等方向的文章。

3. 查看作品  
   访问者希望查看作者在 GitHub 上的项目、自己开发的小工具，以及写过的文章。

4. 理解成长路径  
   访问者希望了解作者如何一步步成为现在的自己，包括开始写作的契机等关键节点。

5. 建立连接  
   访问者希望通过 GitHub、X、邮箱或其他社交媒体，以比较含蓄的方式与作者建立连接。

## 4. 页面结构

### 首页

首页承担第一印象建立任务。

首页需要表达：

- 这是阿川 Blog。
- 阑梦清川是一名 AI 爱好者和内容创作者。
- 这个网站用于了解作者、阅读内容、查看作品。

建议模块：

- Hero：站点名称、作者身份、一句话介绍、核心入口。
- 个人简介预览：简要介绍作者是谁。
- 成长路径预览：展示几个关键节点。
- 作品精选：展示 GitHub 项目、小工具、文章等精选内容。
- 最新文章：展示 AI Agent、读书笔记、AI 实践案例等文章。
- 连接入口：展示 GitHub、X、邮箱和其他社交媒体入口。

### 关于我

用于完整介绍阑梦清川。

内容包括：

- 个人简介。
- 当前身份：AI 爱好者和内容创作者。
- 关注方向：AI Agent、读书笔记、AI 实践案例。
- 核心判断：AI 已经进入 Agent 时代，每个人都需要有自己的网页或网站。

### 学习经历

用于介绍正式教育背景和工作经历。

该页面应偏事实性、清晰、可信，避免包装成厚重的职场简历。

每条经历建议包含：

- 阶段名称。
- 时间。
- 机构或场景。
- 简短说明。
- 相关成果或链接，如果存在。

### 成长路径

用于描述作者如何一步步成为现在的自己。

该页面更强调叙事性和关键节点，例如：

- 开始写作的契机。
- 接触 AI 或 AI Agent 的关键节点。
- 形成持续学习习惯的节点。
- 重要内容创作、项目或实践经历。

### 作品集

用于展示作者的个人作品。

作品类型包括：

- GitHub 项目。
- 自己开发的小工具。
- 自己写的文章。

每个作品条目建议包含：

- 标题。
- 类型。
- 摘要。
- 时间。
- 标签。
- 查看链接。

### 文章

用于承载长期内容创作。

主要内容方向包括：

- AI Agent。
- 读书笔记。
- AI 实践案例。

文章列表需要展示标题、摘要、发布时间、标签或分类。

### 联系

联系方式应保持含蓄，不做强求职或强商业转化。

计划包含：

- GitHub 链接。
- X 链接。
- 个人邮箱。
- 其他社交媒体链接。

具体链接尚未确认前，不得虚构真实账号或邮箱。

## 5. 信息架构

网站的信息架构应围绕“理解作者”展开。

一级导航建议：

- 首页
- 关于我
- 学习经历
- 成长路径
- 作品集
- 文章
- 联系

核心理解路径：

1. 识别：这是阿川 Blog。
2. 定位：阑梦清川是一名 AI 爱好者和内容创作者。
3. 背景：了解正式学习经历和工作经历。
4. 成长：理解作者如何一步步成为现在的自己。
5. 作品：查看 GitHub 项目、小工具和文章。
6. 内容：持续阅读 AI Agent、读书笔记和 AI 实践案例。
7. 连接：通过 GitHub、X、邮箱或其他社交媒体建立联系。

## 6. 目标衡量范围

第一版网站不以复杂增长指标为核心，而以是否完成“个人理解与可信度建立”为衡量标准。

可衡量目标包括：

- 访问者能否在 30 秒内理解作者是一名 AI 爱好者和内容创作者。
- 访问者能否快速找到作品集、文章、学习经历和成长路径。
- 访问者能否明确感受到作者是可信的 AI 深度用户和 AI Agent 资深用户。
- 网站是否能清晰承载 GitHub 项目、小工具和文章。
- 网站是否避免过重求职风格，保持为作者自己而设计的个人站气质。
- 网站是否能持续扩展未来五年的内容。

## 7. 当前不做

第一版暂不做以下内容，除非后续明确补充：

- 强求职导向的 Hire Me 页面。
- 账号系统。
- 评论系统。
- 订阅系统。
- 复杂搜索系统。
- 大量营销式转化模块。
- 未确认的真实联系方式、学校、公司、项目或奖项信息。

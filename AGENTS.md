阿川 Blog 是面向网络用户，用于了解阿川的个人博客或者说网站。

本项目由下面几份核心文档定义，请按照以下顺序阅读：

1. brief.md
   内容是项目长期的纲领、本质用户价值以及边界。
2. prd.md
   内容是首版页面信息架构和目标衡量。
3. design.md
   内容是视觉与 UX 风格，包含 design tokens。
4. architecture.md
   包含技术栈、代码内容、组织结构以及部署。
5. contents.md
   每个内容槽位需要参考的一些相关内容。
6. agents.md
   也就是本文件，包含 AI 入口和协作相关规则。
7. guardrails.md
   针对于交互过程中使用 agent 出现的问题，需要避免的情况请写到该文档下面。


这几份 Spec 文档是 Source of Truth，代码是文档的实现。写代码时如果发现需求或角色变化，需要同步更新对应的文档。

更新规则如下：

1. 改页面结构或信息架构：更新 prd.md
2. 改颜色、字体、组件、状态、动效：更新 design.md
3. 改架构、框架、依赖、部署、数据存储：更新 architecture.md
4. 确认了某个地方的文案：
   把 content.md 对应的内容进行参考。如果触及 brief.md 的边界，先停下确认是否需要修改 brief.md。
5. 针对用户在交互过程中所说的需要更新的 Agent，以及不能再犯的错误、不能做的事情：请更新到 guardrails.md

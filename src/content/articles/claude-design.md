Anthropic 又上线了一个叫做 Claude Code 的东西。发布之后，Figma 应声暴跌了 7.6%。

Claude Code 属于 Anthropic Labs 实验室发布的一款崭新产品，由前段时间 Anthropic 推出的新模型 Claude Opus 4.7 驱动。它几乎囊括了所有的设计流，涵盖了从产品原型设计到演示文稿 PPT，再到官方宣传网页搭建以及产品设计等，几乎所有和 Design 相关的内容。

如果是之前，我们可能还需要打开 Figma 写 CSS，但有了 Claude Code 的加入，我们只需要在 Claude Code 上完成整个设计流程。

这款产品的定位和解决的问题非常直接：
1. 对于设计师：他们没有时间搜索太多的方向。现在设计师可以通过 Claude Code 快速跑出几十个方向，然后再去精挑细选。
2. 对于非设计背景的人（如产品经理、创始人、市场营销人员）：他们有想法但没办法实现。
   (a) 产品经理可以通过 Claude Code 画原型图完成设计，再交给开发。
   (b) 创始人可以直接从项目大纲做出完整的 PPT。

Claude Code 的交互核心是进行 Chat（对话），而不是像 Figma 以及前段时间谷歌的那个产品一样进行拖拽。它更像是和一个设计师聊着聊着就把东西做出来的感觉，所以它在效率上可能会有一个实质性的变化。

说到谷歌的产品，那个叫 Google Stitch（S-T-I-T-C-H）。当时谷歌的 Stitch 设计 Agent 出来的时候，Figma 就跌了一下；现在 Claude 也出来做这样一款设计类工具，传统的设计类公司就更危险了。

废话不多说，我们直接来看一下具体的过程和效果吧。
# 1.效果展示

马上就是 2026 年的世界读书日，而 2026 年恰逢莎士比亚逝世 410 周年。

最近我在小红书上也听了一些和莎士比亚相关的播客，所以我想让他做一个和莎士比亚生平相关的内容。我打算直接描述我的需求，让他自行进行设计就可以了。

首先，我在左边描述一下需求。因为它会给出一些预设选项——毕竟我不太懂设计，它就会列举出一些可能的选项让我进行选择。

这就类似于我们登录一个 App 时，系统会问你：
1. 喜欢什么样的风格？
2. 想要侧重于哪些内容？

然后它会询问我喜欢的风格，以及我设计的这个东西是想侧重于：
(a) 莎士比亚的时代背景
(b) 作品分析
(c) 生活琐事
(d) 他的影响力

它会提供这一系列的预设选项供我们进行选择。
![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com//20260418124823323.png?imageSlim)
中间他会做出一些半成品供我们选择。比如在设计过程中，他会询问并提供很多设计方向，例如：
1. 书卷的形式
2. 剧场的形式
3. 手稿的形式
4. 时间轴的形式

我选择的是时间轴的形式。接着，他会根据我们选择的方向继续进行推演。

针对设计过程中不满意的地方，比如在这个过程中产生了很多占位图，我觉得这些占位图都是线框稿阶段的产物，没有实质性内容，但他可能忘记删除了。这时候我们需要手动地和他 chat，让他删除掉。

你可以通过这些细节处理功能对细节进行修改，都是没有问题的。

# 2.细节功能说明

这次 Claude 提供了一些非常实用的功能，我们将挨个进行介绍：

1. Comment（评论）
2. Edit（编辑）
3. Draw（绘图）
4. Export（导出）

Comment（评论）功能实际上是针对一些细节部分设计的。点击之后，你可以圈中特定区域并要求它进行针对性修改，这样处理起来指向性会更强一些。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com//20260418124359758.png?imageSlim)


Edit（编辑）
实际上就是点击之后，针对设计出来的结果，对其字体、背景颜色等属性进行修改。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com//20260418124422900.png?imageSlim)


DRAW（绘图）实际上，你可以手画一些地方，然后进行修改。

相当于就是通过标注那种方式，让它去执行修改。

关于导出功能，它支持 HTML、PDF、PPTX 等多种格式，方便在其他工具中进行使用。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com//20260418124441041.png?imageSlim)


# 3.总结

Claude Design 目前的目标或定位，就是把从 0 到可以讨论的原型之间的距离和时间缩短。

对于非设计师的小伙伴来说（比如产品经理、独立开发者、内容运营），都可以通过 Claude Design 的帮助做出原型，然后再进行选择。

这解决了传统链路上需要先写需求、再评审、最后才能出稿的问题。现在直接打开 Claude Design，描述需求并迭代修改，基本上就可以做出一个小小的 demo 了。

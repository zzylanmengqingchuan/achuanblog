这篇文章介绍的是一个画图像的 Skill，GitHub 仓库链接如下所示。

我在 Claude Code 里面尝试过，感觉效果还是蛮不错的。

Github地址：https://github.com/moon09300731/drawio-architecture-skill

其实之前我也试过一些画图的 Skill，比如 Obsidian CEO做的这个Excalidraw 绘图以及 Canvas 画布绘图，但我觉得效果都不是很好。

因为总有一些地方我感觉它的控制力度不是很好。比如说 Excalidraw 的那个，虽然手绘风格很好看，但是使用智谱的这个模型时，感觉内容经常重叠在一起，视觉效果不好。至于 Canvas 画布，感觉也没有那么满意，没有体现出对应的特色。

这个仓库虽然也是画图的，但它支持 Draw.io 的架构图、流程图和系统设计图，而且支持导出 PNG 和 PDF 这样的文件格式，这是我很看重的一点。

因为如果是做 UI 内容的话，通常还需要使用特定的工具才能打开，而且效果不是很好。我主要是看中了它对 PNG 和 PDF 这类图片或文档格式的支持，我觉得这很不错。

昨天下午我试了一下，让它生成 PNG，效果如下所示，感觉还很不错。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260403143616200.png)
我觉得还很不错吧，起码比我想象的要好。

然后整个过程也就是在 Frosk 里面去完成的，使用的是智谱的智谱清言 GLM5.1 最新的这个模型。

现在的 Skill 也非常多，也不能说是劣币驱逐良币吧，只能说各种乱七八糟的都往这上面靠。其实有一些从某种程度上来说，可能也违背了 Skill 的初衷。

有些其实就是一个提示词，或者是 Markdown 语法，并不能完全算是一个 Skill。但是毕竟是热点嘛，所以很多人不管三七二十一，反正就往 Skill 上面靠，咱也没有办法。

这个安装使用非常简单，你就打开之后让它阅读这个项目，然后让它去给你安装依赖、MCP 之类的。

这些都是可以让 Agent 自己去完成的。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260403143800469.png)

它安装的时候会告诉你安装了哪些内容，然后告诉你如何使用。

然后你按照它的指令开始使用就可以了。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260403143913767.png)
智谱的这个模型自带搜索功能，虽然搜索效果比较一般，甚至可以说有点拉胯，但确实是可以用的，只是耗时可能长一点。

我尝试让它搜索了一部最近上映的电影信息，并生成人物关系图谱。它原本想生成 Draw.io 格式，但我觉得那种格式查看起来不方便，效果也一般。

因为模型支持 PNG，所以我让它生成 PNG 图片。不过生成 PNG 需要运行脚本，它是调用 Python 里的 matplotlib 库来生成的，这就需要检查 Python 的安装环境，可能存在一定的门槛。

整个过程如下：
1. 模型会自动编写 Python 脚本代码并运行。
2. 运行成功后，它会提示你可以使用电脑自带的图片查看工具。
3. 图片里会清晰地梳理出角色、关系以及相关的注释介绍。

我觉得整体的完成度还是非常高的。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260403144058597.png)

虽然这个 skill 所在的 GitHub 仓库是两个月之前的，但确实没有什么知名度。我也是偶然间发现的，就想试一试。

结果我发现它比我之前用的 Obsidian 的 skill，以及 excalidraw 的 SQL 都好用太多了。所以我觉得：

1. 不一定那些 Star 数量多的项目就一定好用。
2. 也不一定这种 Star 数量很少、没什么人关注的项目就一定不好用。

这就是一个典型的例子：它只有四个 Star，用的人也比较少，开发者本身可能也没什么流量，但是人家这个 skill 用起来确实很不错。所以我推荐大家也去尝试使用一下。

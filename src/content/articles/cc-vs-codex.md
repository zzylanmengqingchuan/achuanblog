这篇文章来聊一聊我自己 vibe Coding 对于两款工具的选择。这两款工具也是目前市面上用得最多的两款工具：Claude Code 和 CodeX。

1. Claude Code 属于 Anthropic
2. CodeX 属于 OpenAI

这两款工具属于两家不同的厂商，但也是目前最受欢迎的产品。它们在开发者当中占据了大头，是使用最多的两款产品，我认为也应该成为你 Web Coding 的首选。

其实关于 Claude Code，我之前已经聊过很多次了。这次主要是再详细地说一下，以及它和 CodeX 之间是如何选择的。

# 1.前言

我使用 Claude Code 已经半年的时间了，但使用 CodeX 还不到一个月，所以我会对 Claude Code 更加熟悉一些。

不过，当你熟悉了一款工具后，去使用另外一款工具其实也是很容易的。因为这两款工具的使用门槛并不高，主要是在环境配置上面，可能会难倒很多人。

# 2.我需要claude账号么

虽然我在之前的文章中多次强调，想要使用 Claude Code，你不一定需要 Claude 官方的账号。但是仍有无数的小伙伴依然存在这样的误区，认为自己必须拥有一个 Claude 账号，其实这当然是不需要的。

Claude Code 这个工具是任何人、任何用户、在主流操作系统上（windows and mac)都是可以安装的。

如果你是新用户，想要进行安装，我推荐下面这个教程：

# 4.为什么推荐 VS Code 插件？

上面你有了自己的套餐，安装了 Claude Code，其实就已经可以在命令行里面去使用了。

但我依然更加推荐 VS Code 插件里面的这个方式，这也是我之前文章多次提到的。这个插件的名字叫做 Claude Code for VS Code。

安装之后，它就会内嵌在你的代码编辑器里面。

你可以左边看着项目文件，右边再使用这种对话的形式，让 Claude Code 去修改你的代码，畅快地进行vibe Coding。这是我比较推荐的一种方式。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260328212951735.png)
这个插件我已经使用了半年，而且在使用的过程中，能明显感觉到它自身在不断升级。它应该经历过好多次更新，设计也在一直发生变化，所以是有人一直在维护着的。

主要是很多小白对于终端或者说命令行这种方式本来就不熟悉。安装一个插件其实和命令行效果是一样的，但是使用这种图形化界面的方式，我觉得对于大部分的新手更加友好一些。

# 5.codex如何结合

上文也说了，我使用 Claude Code 接近半年的时间。这个月我有一个偶然的机会，可以免费使用一个月的 plus，所以也是第一次体验到了 CodeX 这款工具的用法。

我最开始是直接下载了 CodeX 的客户端工具。它有一个专门的客户端，在里面使用时可以打开多个线程，实际上就是同时打开多个项目，这样切换起来也更加方便。

我左边就打开了六七个项目，可以更加自如地进行切换。不过它的一个缺点是，虽然你打开了项目，但不能像 VS Code 里面那样直接看到项目的具体内容。

这其实是有利有弊的：
1. VS Code 的特点：
   它通常只能打开一个项目，你能看到完整的文件结构，但想切换到其他项目时就不是很方便。
2. CodeX 的特点：
   它虽然不能看到文件的具体内容，但你可以同时打开多个项目，切换效率更高。

所以这两者之间确实是各有千秋。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260328213321535.png)
当然，这个 CodeX 也支持在一些代码编辑器里面打开。你使用过的话，就会知道它的右上角有一个打开选项，支持一些你已安装的代码编辑器。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260328213454076.png)
可能你会问这两者到底有什么差别？

其实就我个人使用场景而言，主要的差别就是多模态能力。智谱（GLM）作为一个国产模型，目前是不具备多模态能力的。

我在写代码的过程中，有些地方需要通过截图来跟它说明原因或者问题。这个时候，如果你使用 Claude Code 加智谱的 GLM 模型，它是无法识别图片里的错误的。你只能去描述错误，或者复制一些报错信息。虽然这种方式也能解决问题，但如果你想直接上传图片进行说明并让它修改，它由于不具备多模态能力，就无法理解你的需求。

在这种情况下，我就会选择 CodeX。CodeX 的模型具备图像识别能力，很好地解决了我这方面的问题。

我之前一直使用 CodeX 的客户端工具。今天我发现，其实 VS Code 里面也是有插件的，而且用起来也很不错。

这样的话，你可以安装两个插件，根据你自己的场景来选择合适的工具使用。

你安装 CodeX 这个插件之后，你的历史记录会全部同步过来，和客户端工具里的使用方法是一模一样的。只不过在这种方式下，进行项目之间的切换就没有那么方便了。

这个区别我在上文也已经提到了。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260328213738669.png)

从我个人的体感上来说，其实 Claude Code 加上智谱模型这种方式，它的体验会更好一些。这里说的体验主要是指速度和响应上会更快。

而 CodeX 每次建立连接需要一定的时间，整个过程其实没有 Claude Code 那么快。这只是我个人的一个感觉。

当然，CodeX 是支持 Standard 模式和 Fast 模式的：
1. Standard 模式（标准模式）：这种情况下速度比较慢。
2. Fast 模式（快速模式）：它比标准模式更快一些，但是积分的消耗也更快。

所以这一块，你自己斟酌进行权衡即可。

# 6.Git

 属于进行代码版本控制的一种工具。

在 Vibe Coding 的过程中，我认为它的重要性是不言而喻的。因为 AI 在修改代码时，可能会出现一些问题，甚至出现越改效果越差的情况。

这时候版本管理的作用就凸显了出来：
1. 如果你会进行版本管理，你可以随时回退到一个指定的版本。
2. 但如果你没有进行版本管理，处理起来可能就会稍微复杂一些。

所以，在 Vibe Coding 的过程中，版本管理的重要性是不言而喻的。

你可能又要说，我不懂 Git 指令什么的，我也不太会。但是我觉得这些都不是大问题。

在没有 AI 的时代，你可能需要学习指令，当然也有一些桌面端的工具。但是 AI 的出现其实大大降低了这件事情的门槛，你只需要了解一些基本的概念，甚至你不了解也是可以完成这件事情的。

我们可以把自己的代码推送到一些代码仓库。

国际、国外的话，比如说 GitHub；国内的话，比如说 Gitee，都是很不错的选择。

下面我介绍三种推送代码的方式：

1）古法推送，也就是自己书写代码的方式。

借用这个词，我来介绍第一种推送方式，即“古法推送”：你自己去写代码，然后添加到暂存区、提交、最后推送。这种就是传统的命令行方式。

2）VS Code 插件推送

这是我之前使用最多的方式：在 VS Code 里面和远程建立连接，通过图形化界面的方式进行推送，不需要写任何一条指令。

针对于代码的修改，目前我这里显示是空白的，所以没有什么需要提交的内容。

如果你修改了代码，它就会把所有改动自动添加到 Git 管理面板中。你可以直接进行以下操作：
1. 添加到暂存区
2. 提交（Commit）
3. 推送同步（Push）

这些全部都是点击操作，不需要写任何一条指令。

![image.png](https://zzy-1326340203.cos.ap-beijing.myqcloud.com/20260328214453163.png)
针对你的每一次 Push，它下方的蓝色时序表示的就是你记录的每一次推送。

这也是我为什么要强调代码管理。比如你让 AI 连续写了两次代码，最后它越改越差，你可以直接指定让它回退到对应的版本。这个方法我试了很多次，在 Vibe Coding 的时代是很有用的。

3）可能你觉得第一种方法太难了，第二种方法好像仍然很费劲，那么我可以告诉你第三种方法，就是让 Claude Code 去帮你完成。

下面就是我的截图。你可以直接让 AI 帮你推送代码，只需一句自然语言描述，它就会自动执行一些脚本。

虽然你会发现它实际执行的也是 add、commit 和 push，本质上没有太大区别，但这种方式的意义在于：即使你不会 Git 指令，也能在 AI 的辅助下完成代码管理。

# 7.总结

这篇文章简单地介绍了一下 Claude Code 以及 Codex 之间的对比。

Claude Code 是我使用了半年多的一款工具，而 Codex 我使用不到一个月。今天这篇文章本来只想分享一下 Codex 的相关内容，没想到还是写了这么多，也介绍了一些我最新的心得体会。

比如 Codex 的这个插件，实际上今天我也是第一次使用。之前我其实并不是很在意版本管理这件事情，但是最近发现，去进行代码的版本管理还是非常有必要的，也是非常重要的。

当你真正具备了这种能力，GitHub 上无数的开源项目都可以为你所用。你可以在上面找到自己感兴趣的项目，然后真正利用起来，在 AI 的帮助下去学习。这在几年前基本上还是少数人的专利，但是现在，即使你没有编程的基础，你也可以在 AI 的帮助下去实现这些事情。

你只需要拥有一个好的 idea，那么一切或许没有你想象中那么困难。
